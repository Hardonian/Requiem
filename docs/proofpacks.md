# Proofpacks

A **proofpack** is a first-class, self-validating execution receipt that
cryptographically binds all aspects of a Requiem execution.

## Schema (v2)

```json
{
  "version": 2,
  "execution_id": "run_abc123",
  "input_hash": "<blake3-64hex>",
  "policy_hash": "<blake3-64hex>",
  "workflow_hash": "<blake3-64hex>",
  "tool_call_hashes": ["<blake3-64hex>", ...],
  "state_hash": "<blake3-64hex>",
  "cas_references": {
    "output.txt": "<blake3-64hex>"
  },
  "merkle_root": "<blake3-64hex>",
  "timestamp_logical": 42,
  "timestamp_iso": "2025-01-01T00:00:00.000Z",
  "signature": "<ed25519-hex>",
  "signing_status": "signed|unsigned",
  "engine_version": "1.3.0",
  "hash_version": 1
}
```

## Fields

| Field | Description |
|-------|-------------|
| `execution_id` | Unique identifier for the execution |
| `input_hash` | BLAKE3 hash of canonical request inputs |
| `policy_hash` | BLAKE3 hash of policy rules at evaluation time |
| `workflow_hash` | BLAKE3 hash of workflow definition |
| `tool_call_hashes` | Ordered BLAKE3 hashes of each tool invocation |
| `state_hash` | BLAKE3 hash of final execution state |
| `cas_references` | Map of output filenames to CAS digests |
| `merkle_root` | Merkle root covering all hashes above |
| `timestamp_logical` | Logical timestamp (not wall-clock) |
| `signature` | Ed25519 signature over merkle_root |

## Invariants

1. **Self-validating**: `merkle_root == compute_merkle_root(all_digests)`
2. **Immutable**: Once generated, fields cannot be modified without invalidating `merkle_root`
3. **Deterministic**: Same execution always produces same proofpack (excluding `timestamp_iso`)
4. **Verifiable**: `requiem verify proofpack.json` returns VALID or INVALID

## Verification

```bash
# Verify a proofpack
rl verify proofpack.json

# Output: VALID or INVALID with detailed check report
# Checks performed:
#   - Version compatibility
#   - Hash format validation (64-char hex)
#   - Merkle root recomputation
#   - CAS reference format
#   - Execution ID presence
```

## Generation

Proofpacks are generated by the proof engine after execution:

```typescript
import { buildProofpack, saveProofpack } from '@requiem/proofs';

const pack = buildProofpack({
  execution_id: 'run_abc123',
  inputs: { tool_id: 'echo', data: 'hello' },
  policy_rules: [{ rule_id: 'allow_echo', effect: 'allow' }],
  workflow_definition: { steps: ['step_1'] },
  tool_calls: [{ tool_id: 'echo', input: { data: 'hello' }, output: { result: 'hello' } }],
  final_state: { completed: true },
  cas_outputs: { 'output.txt': '<cas-digest>' },
  timestamp_logical: 1,
  engine_version: '1.3.0',
});

saveProofpack(pack, 'proofpacks/run_abc123/proofpack.json');
```

## Storage

```
proofpacks/
├── latest/              # Most recent proof run
├── determinism/         # Determinism test artifacts
└── <execution_id>/      # Per-execution proofpacks
```
