# Archive Index

This index tracks archived documentation so history remains discoverable without polluting the canonical doc surface.

## Archive groups

### `docs/archive/root-reports/`

Archived repository-root status and planning reports moved out of the root surface.

Reason:

- These files were mostly point-in-time reports, strategy explorations, or implementation scratch summaries.
- Keeping them at repository root made canonical onboarding harder.

Canonical replacements:

- Project front door: `README.md`
- Onboarding: `docs/GETTING_STARTED.md`
- Architecture: `docs/ARCHITECTURE_OVERVIEW.md`
- Governance: `docs/DOCS_GOVERNANCE.md`, `GOVERNANCE.md`

Archived files include:

- `BLOCKERS_CLOSED.md`
- `CATEGORY_COLLAPSE_REPORT.md`
- `FINAL_SHIP_REVIEW.md`
- `QUICKSTART.md`
- `READYLAYER_CLI_IMPLEMENTATION.md`
- `RELEASE_CONFIDENCE_REPORT.md`
- `REMAINING_NONBLOCKERS.md`
- `ROUTES.md`
- `TEST_DATA_FOUNDRY_IMPLEMENTATION.md`
- `TOP_5_EXISTENTIAL_WEAKNESSES.md`
- `TYPE_ERROR_RESOLUTION.md`
- `WHAT_CHANGED.md`

## Archive policy

All future archive moves should:

1. Preserve files under `docs/archive/...`.
2. Add/update an entry in this index.
3. Link to canonical replacements.
