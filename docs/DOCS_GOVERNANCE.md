# Documentation Governance

## Canonical docs

- `README.md` — product overview and entry links
- `docs/GETTING_STARTED.md` — first setup path
- `docs/REPO_STRUCTURE.md` — where things live
- `docs/API_GOVERNANCE.md` — API contracts and route classes
- `docs/VERIFIED_CLAIMS.md` — executable proof mapping

## Archive policy

- Historical reports and one-off audit artifacts go under `docs/archive/`.
- New docs must be one of: canonical, reference, runbook, or archive.
- If a doc becomes superseded, move it to archive and link it from `docs/ARCHIVE_INDEX.md`.
