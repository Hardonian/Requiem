# Roadmap

This roadmap captures current direction at a high level. It is intentionally concise and updates as priorities change.

## Near term

- Improve canonical documentation and repository governance.
- Strengthen deterministic verification and boundary checks in CI.
- Reduce duplicate or stale docs in public project surfaces.

## Mid term

- Improve developer ergonomics for Reach CLI local workflows.
- Expand architecture and operational runbook clarity.
- Continue hardening auditability and evidence generation paths.

## Ongoing

- Keep OSS and enterprise boundary explicit in code and docs.
- Maintain release hygiene through changelog and verification evidence.

For shipped changes, see [CHANGELOG.md](./CHANGELOG.md).
