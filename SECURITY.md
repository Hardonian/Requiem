# Security Policy

## Reporting a vulnerability

Please do not report vulnerabilities in public issues.

- Email: **security@ready-layer.com**
- Include: affected component, reproduction steps, impact, and any proof-of-concept details.

## Response targets

- Acknowledge report within 3 business days.
- Provide triage status after initial review.
- Coordinate disclosure timing with reporter when possible.

## Scope notes

This repository contains open-source components and integration code. Hosted enterprise incident handling for ReadyLayer Cloud is managed through enterprise support channels.

## Security expectations for contributors

- Do not commit secrets or private keys.
- Avoid including sensitive operational/customer data in docs or tests.
- Use least-privilege defaults for new features.
