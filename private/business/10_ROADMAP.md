# Product Roadmap: Requiem

## 1. 30-Day: Launch & Stability

- [ ] **Public OSS Release**: Version 1.0.0 on GitHub.
- [ ] **Stability Pass**: Harden the C++ core for Windows and Linux edge cases.
- [ ] **Reach v1.0**: Complete CLI help documentation and `reach doctor` suite.
- [ ] **Initial Policy Templates**: Standard "Safe Tooling" policies for common MCP servers.

## 2. 60-Day: Ecosystem & Integration

- [ ] **ReadyLayer Pro Beta**: First multi-tenant cloud dashboard.
- [ ] **MCP Autodiscovery**: Automatically import and govern third-party MCP tools.
- [ ] **The "Explain" Engine**: Enhanced `reach explain` using local LLMs to describe deterministic traces.
- [ ] **Cost Accounting v1**: Per-tenant spend tracking and budget enforcement.

## 3. 90-Day: Enterprise Readiness

- [ ] **Signed Artifact Chain**: End-to-end signing for high-compliance customers.
- [ ] **Cluster Sync**: Real-time policy distribution across multiple Requiem nodes.
- [ ] **Advanced Drift Detection**: Automatic alerting when semantic drift exceeds a defined threshold.
- [ ] **VS Code Extension**: Direct integration of `reach` verification into the IDE.

## 4. 6-12 Month Themes (Future Vision)

- **Universal Replay**: Interoperability with other agent frameworks (LangChain, AutoGPT) via bridge adapters.
- **Hardware-Backed Receipts**: Leveraging TEEs (Trusted Execution Environments) for even stronger proofs.
- **Requiem Marketplace**: Shareable, verified policy sets and "Safety Skills" for AI agents.

## 5. Alignment Disclaimer

*This roadmap is based on current repository trajectory and contributor capacity. Priorities may shift based on community feedback and enterprise requirements.*
