# Talent Assessment & Performance: Requiem

**Version**: 1.0.0  
**Last Updated**: 2026-03-02

## 1. Performance Dimensions

We do not use generic performance metrics. Employees are evaluated on their contribution to **System Coherence**.

### Dimension 1: Technical Invariants

- Did the employee introduce non-determinism or flakey tests? (Goal: 0)
- Did they reduce technical entropy in the modules they touched?

### Dimension 2: Security & Integrity

- Did they identify a potential sandbox escape or policy bypass?
- Did they maintain the privacy-first data handling standards in their features?

### Dimension 3: Operational Velocity

- How many "Verified Milestones" were closed on time?
- Quality over quantity: Did their code require rework to meet the "Green" objective?

## 2. Review Template (Quarterly)

- **The Big Win**: What was the most significant contribution to the Requiem moat?
- **Entropy Identified**: Where did you find complexity, and did you eliminate it?
- **Goal Alignment**: Update on 6-month OKRs.

## 3. Interview Rubric (Systems Role)

- **Coding**: Implement a simple hash-based state machine.
- **System Design**: Design a sandbox that allows network access but prevents data exfiltration.
- **Values**: "Why does determinism matter in a world of probabilistic AI?"

## 4. Onboarding Checklist (The First 7 Days)

- [ ] Run `reach doctor` and fix any environmental drift.
- [ ] Submit a PR that adds a new invariant test to the native engine.
- [ ] Generate a cryptographic receipt for a custom agent run.
- [ ] Read the *Deterministic AI Compute* whitepaper.
